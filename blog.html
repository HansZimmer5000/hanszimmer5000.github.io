04. Jun 23 - Email Signature Banner

A colleague once created very handy signature banners, stating the usual info like name, role and email. Back then, proprietary software was used, probably something along the lines of Photoshop. Today I thought to myself, if it is possible to create the banners also via OSS to automate and enable less restricted access to the creation of the banners.
 The language to use was quite clear from the start. The PoC nature of this project screams "PYTHON!". To help with the image creation, I swiftly found Pillow. After a bit of searching around, looking through documentation and multiple Stack Overflow messages, I ended up with:


Creating the script was pretty straight forward. Overall, it took only 1.5 hours. And additionally to speed, the quality also fits, the resulting image is pretty close to the original. To make it even closer, and to make it ready for different names and role names, small changes are left to do. Note that the image shown above is an example.
Regarding the code, take a look at my python script. I do not think that the usage of Pillow needs explanation, so the following notes will cover more general elements:
- Must know to run the script:
  • With the current script, a local TTF file is needed to provide the font. On MacOS I could extract the TTF file from the already installed "Font Book" application.
    •
- Should know to adjust the script:
  • To get the RGB colours, I used the already installed "Digital Colour Meter" application. Take the output with a grain of salt. In my case the values shown were 2 to 20 below the actual RGB values, and even with that in mind the colour was not exactly the same. However, it was close enough.
    •
  • The script is logically split in four parts. First, the constants are defined, to make it easy for adjustments. After that, the left and right images are created. This separat creation is intended to enable easier changes to content and style. At last, the images are put together.
    •
  • I didn't instantly find a native function to align the text to the right side (e.g. for first name). So, a simple calculation was implemented to get the desired X-axis value: (image width) - (text length) - (padding). Subtracting the padding is used to have some space between the image right border and the last character.
    •
- Nice to know:
  • Due to the ad hoc "let's try this one" nature, I didn't put much emphasis on readability of the code (e.g. everything in one function, different ways of defining values). In case it is used at work, the readability is probably the first task I would tackle to make sure other colleagues can also maintain / adjust it.
    •
To conclude, Pillow was good fun and lead to a usable and flexible result quickly. I highly recommend it for similar use cases. I already have another use case in mind for which Pillow could be useful, let's see.

23. Apr 23 - ChatGPT with Bash

ChatGPT does not need an introduction at this point. I tried it out recently to create a tail recursive script in bash to get something for me from the Azure Portal. So, I thought maybe ChatGPT can help with providing the close-to-final code directly. Saving me some time. ChatGPT 3.5 was used in version Mar 23.
So, I initially asked: Can you write a tail recursive bash script for me, that flattens a JSON list that may contain JSON lists
In the end I wanted to get a script, that reads through a JSON list. If an JSON object contains a certain key and value, it should go into recursion. Otherwise, extract certain values. As the nested lists may contain the same JSON object (and so id) multiple times, only the first time should it be looked at, to save time and duplicates in the final result.
After more prompts, I slowly but surely got ChatGPT to generate the following script: scriptv1.sh
At first glance, great! I especially like the comments it put into the script. However, at a closer look certain parts are weird:

Line No. Criticality What I find weird How I would address it
4 Nice

Use of "function <name>" instead of "<name>()", e.g:

function flatten_json_list {
 Personal preference. Fyi: "()" is POSIX, see also GNU Documentation and Googles StyleSheet. For example:
flatten_json_list()
7 Must Variable "ids" is set, but never used nor returned. This is important, because this variable represents the ids that were already looked at. Return the "ids" variable, which is easier said than done. Like in Python, babyeierleicht", you just return two values. This is way harder with Bash as bash only can return text and only one value. What can be done is to return a JSON object that contains the "result" variable and the "ids" variable. Which then gets parsed again one level higher to make use of the values. If you think you can also implement that via a global bash variable: nope, as subshells cannot change variable from parent shells
9,16 Must The '== "object"' and '=="array"' checks always fail. For Example:
$(echo "${item}" | jq 'type') == "object"
 Why this is I am not sure. However, one can make use of jq features to check for certain values: jq 'type=="array". For Example:
"$(echo "${item}" | jq 'type=="object"')" = "true"
16 Should If clause is overall hard to read / understand due to so many things happening in one line:
if [[
  $(echo "${item}" | jq 'type') == "object" &&
  $(echo "${item}" | jq '.type' | grep -q -E '[^[:space:]]') &&
  $(echo "${item}" | jq '.id' | grep -q -E '^[0-9]+$')
]]; then
 Fix is two folds. One is to replace the grep with a jq null check: jq '.type==null'. The other one is to split the evaluations up. For Example into their own each variable that is evaluated before the if. The if then "only" combines the values. 
if [[
  "$(echo "${item}" | jq 'type=="object"')" = "true" &&
  $(echo "${item}" | jq '.type==null') == "false" &&
  $(echo "${item}" | jq '.id==null') == "false"
]]; then
31 Should If input is not given via stdin, cat will block the terminal Replace simple cat with small logic to check if stdin exists or not. If it exists, use it, otherwise try to get input via positional argument(s). For example:
if [ -t 0 ];
  then json_list=${1:?insert json}
  else json_list=$(cat)
fi
34 Nice (Nit-picky) Not possible to provide initial set of known ids. Ids are initially always empty: 
local ids=()
 Have a function before the "real" recursive one, to set a default or the given ones. For example:
flatten_json_list(){
  flatten_json_list_ "{"ids": [],"result": []}" "$1"
}
flatten_json_list "<already known ids from user input>"

After I implemented these and other small improvements, my final script is scriptv2.sh, fully working as intended. See here the diff of the two files.
As a conclusion, ChatGPT with this first test was very good at giving a general idea / structure to solve a problem. However, Bash knowledge is essential to make the script work, and to make sure the script is readable / understandable.

15. Apr 23 - Farming Simulator Equipment Efficiency Calculator

Note that this site is an exemption to the "No JavaScript" rule of my homepage to facilitate the calculation logic.
In the game you may have the situation in which you want to change your equipment. To become newer or, most certainly, bigger. But then you might ask yourself which combination of machines is the most effective? Is it more effective to have machines that can each do one job only but very well (e.g. large working range)? Or rather one machine that does multiple things at a time but all a bit bad (e.g. small working range)?
This calculator helps you to answer the questions and additionally gives an outlook how much time you roughly need with which machine combination. Please input the numbers below as they are in the game.

Beware:
1. add decimal numbers with "." (e.g. "3.2")
2. only as long as all inputs have the same system (e.g. all inputs speed km/h), the efficiencies can be compared. Different systems lead to different numbers resulting in a different efficiency number.
3. if a given equipment (combo machine) is doing two things at a time, only fill in the equipments width and speed once. Example: Seeder that also Fertilize, insert "Seed" data and let "Fertilize1" data empty.
4. the efficiency of one combo machine can be compared to the efficiency of a set of machines, by comparing the efficiency of the combo machine with the "total efficiency" of the machine set. The "total efficiency" does not include time for changing equipment.
5. as of now the field related calculations must be done with "km/h" and "meter" based inputs.

24. Dec 19 - Building this Website

I always wanted to have my own website, for experimenting with this 'Web Dev' stuff and also for presenting myself to the world. The target of this website is to be as simple as possible, among other sources, I was inspired by fefe and his rather simple web blog. I want to go one step further and only use HTML and CSS. This of course limits the visual 'wow'-effect, but that's OK. After I got my domain, I needed webhosting. First I tried DigitalOcean, they are good and not free. I will keep them in mind for later projects but for my low requirements GitHub Pages, for free, seems good. The tutorial is very good, even though there is not much to do in the first place. Only the DNS is problematic as no nameserver IPs are listed, weird. My registrar had a support site specifically for GitHub where I could find these IPs. Right now it seems working smooth. Let's see how this turns out in the future.
